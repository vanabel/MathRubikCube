# 🎓 位置 vs 方向：魔方的两个维度

## 🎯 重要概念

魔方有**两个独立的维度**：

### 1. 位置（Position / Permutation）
块在魔方中的**物理位置**

### 2. 方向（Orientation / Twist）
块的**旋转朝向**
- 角块：3种可能的方向
- 棱块：2种可能的方向

### ✅ 已实现
- **位置置换** - 完整实现
- 块的位置变化
- 循环分解

### ❌ 未实现
- **方向/朝向** - 尚未实现
- 角块的翻转
- 棱块的翻转

## 🔬 案例分析：F R U R' U' F'

这是一个经典的**OLL公式**（Orientation of Last Layer）。

### 在真实魔方中

**位置变化：**
- 会改变一些块的位置（包括下层）
- 我的模型显示：(2 7) (3 4) ✓ 正确

**方向变化（未实现）：**
- 主要改变顶层块的朝向
- 使顶层颜色朝上
- **这是OLL的主要作用**

### 在我的模型中

只能看到**位置变化**：
```
角块循环: (2 7) (3 4)
  - UBL(2) ↔ DBR(7)
  - UBR(3) ↔ DFR(4)
```

**看起来影响了下层**，这是正确的！因为：
- F操作物理上会转动DFR和DFL
- R操作物理上会转动DFR和DBR

## 🎲 真实魔方的F操作

```
前面视图：
  1---0      F操作后这4个角都会移动：
  |   |  -->  UFR(0) -> DFR(4)
  |   |       UFL(1) -> UFR(0)
  5---4       DFL(5) -> UFL(1)
              DFR(4) -> DFL(5)
```

✓ **是的，F操作会影响下层前面的两个角块！**

## 💡 为什么会有误解？

### 1. OLL vs 位置

**OLL公式**的主要作用是改变**方向**，而不是位置：
- 在解魔方时，你关注的是颜色朝向
- 位置的改变是"副作用"

### 2. 实际解魔方时

执行 `F R U R' U' F'` 后：
- **视觉上**：顶层某些颜色朝向改变了
- **位置上**：确实有一些块交换了位置
- **通常不注意位置**：因为还没到调整位置的步骤

## 📖 标准魔方记法中的OLL

**OLL（Orientation of Last Layer）：**
- 第3步（CFOP方法）
- 目的：使顶层所有块的**颜色朝上**
- 不关心位置是否正确

**PLL（Permutation of Last Layer）：**
- 第4步
- 目的：调整顶层块的**位置**
- 此时方向已经对了
